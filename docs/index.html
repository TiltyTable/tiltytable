TiltyTable

A reprogrammable tilting ball game.

Imagine the wooden ball maze game, but the holes and walls can move. TiltyTable is a modular, 3D-printed game platform with a tilting table, moving tiles, LEDs, arcade roller-ball control, and Azure Kinect tracking.

Overview

TiltyTable is an open-source hardware/software project for building physical ball games on a reconfigurable table. Instead of one fixed maze, the platform can change the board while someone is playing: walls can become holes, tiles can drop out, and LEDs can tell the player what is about to happen.

System

  • The table tilts using three NEMA-25 motors in a Stewart-platform layout.
  • The player input is an arcade roller ball, not a joystick.
  • Each tile has an individual servo motor that controls its up/down position.
  • An Azure Kinect looks down at the board for ball tracking and automatic calibration.
  • The whole system is designed to be 3D printed, modular, and easy to assemble.

Game ideas

  • A classic point-A-to-point-B maze where the board changes while you play.
  • Tiles that suddenly turn from walls into holes, with LEDs warning the player first.
  • A snake-like survival mode where every tile the ball touches drops down afterward.
  • More short arcade-style games built on the same moving-tile platform.

Current status

  • Stewart-platform motion tests are running on the bench.
  • Azure Kinect color/depth capture, ball tracking, and calibration code exists.
  • Roller-ball input is wired into the control path as a USB input device.
  • Moving-tile modules, servo control, and LED feedback are being integrated.

Hardware

  • 3 NEMA-25 motor Stewart platform
  • Servo-driven moving tiles
  • Arcade roller ball input
  • Azure Kinect overhead sensor
  • Addressable LED feedback

Software

  • Python Kinect web control
  • OpenCV ball tracking and calibration
  • USB input capture
  • Arduino motor, servo, and LED commands
